EXECUTIVE SUMMARY
================================================================================

The redirect loops and blank pages are NOT caused by authentication failures
or API communication issues. The problem is straightforward:

    THE HOME PAGE (page.tsx) IS INCOMPLETE

    It has code to redirect unauthenticated users to /login
    But it has NO CODE to show content for authenticated users

    Result: Authenticated users see a blank page with
            "Redirecting to login..." message

This is not a bug - it's an unfinished feature in an ongoing CRA to Next.js
migration.

================================================================================
ROOT CAUSE IDENTIFIED
================================================================================

File: /web-next/app/page.tsx
Lines: 18-23

The function ALWAYS returns the same div regardless of authentication state:

    return (
      <div style={{...}}>
        <p>Redirecting to login...</p>
      </div>
    );

What should happen:

    if (loading) {
      return <LoadingSpinner />;
    }

    if (!user) {
      router.push('/login');
      return null;
    }

    // User is authenticated - show content!
    return <Dashboard user={user} school={school} />;
