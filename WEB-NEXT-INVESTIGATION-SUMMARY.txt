API COMMUNICATION - DETAILED TRACE
================================================================================

When user HAS token, the following happens:

1. Browser: GET http://localhost:3000
   └─ Receives HTML with <script> for client bundle

2. Client: React hydrates
   └─ AuthContext useEffect runs (line 43-77 in AuthContext.tsx)

3. AuthContext checks:
   const token = sessionStorage.getItem('sf_token') || localStorage.getItem('sf_token')
   └─ If token exists, proceeds to step 4
   └─ If not, sets loading=false and returns

4. API call:
   auth.me()
   └─ This calls: axios.get('/auth/me')
   └─ API client interceptor adds header:
      Authorization: `Bearer ${token}`
   └─ Full URL: http://localhost:5000/api/auth/me

5. Backend:
   Receives: GET /api/auth/me
   Headers: Authorization: Bearer eyJ...
   ├─ Validates JWT token
   ├─ If valid: Queries database for user
   ├─ Returns: { user: {...}, school: {...} }
   └─ Status: 200 OK

6. Frontend receives response:
   AuthContext.tsx line 56-59:
   .then((data) => {
     setUser(data.user);
     localStorage.setItem('sf_user', JSON.stringify(data.user));
     persistSchool(data.school);
   })
   └─ Sets React state with user data

7. Component re-renders:
   page.tsx now sees:
   - loading: false (from finally block, line 70-71)
   - user: {id, name, email, ...}
   └─ Condition: if (true && false && true) = FALSE
   └─ No redirect

8. Output:
   Still renders:
   <div>Redirecting to login...</div>
   └─ User sees blank page ❌

═══════════════════════════════════════════════════════════════════════════════

When API call FAILS (401, network error, etc):

AuthContext.tsx line 61-69:
.catch(() => {
  sessionStorage.removeItem('sf_token');
  localStorage.removeItem('sf_token');
  localStorage.removeItem('sf_user');
  localStorage.removeItem('sf_school');
  setUser(null);
  persistSchool(null);

.finally(() => {
  setLoading(false);

API Response Interceptor (client.ts, line 20-31):
